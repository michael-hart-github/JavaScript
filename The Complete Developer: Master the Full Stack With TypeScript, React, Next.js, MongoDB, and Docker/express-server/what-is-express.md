# What is Express.JS?

- Express is a FOSS backend framework
- Express is built on Node.JS
- Express is (at the time of the book's writing) the de facto standard server framework for the Node "ecosystem"
- Express is (at the time of the book's writing) foundational to full-stack web dev work

# What does Express do?

- Express is designed to build WebApps and APIs
- Express "offers middleware" used by HTTP servers for things like:
  - Caching
  - Content negotiation (?)
  - Cookies
  - Cross-origin requests (?)
  - Redirects
  - Etc. (Very helpful...)

# Who uses Express?

- Full-stack web devs who run backend frameworks
- Devs who work in the Node "ecosystem"
- Devs who work on FOSS projects

# What other packages use Express?

Next.JS has a built-in server that "borrows heavily" from Express. However, Next abstracts the middleware mentioned above on the developers behalf, so they do not need to directly interact with the server.
